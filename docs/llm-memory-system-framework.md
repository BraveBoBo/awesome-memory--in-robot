# 大模型记忆系统分析框架

本文整理一套可用于分析 LLM Agent 与具身机器人记忆系统的统一框架。概念来源参考 [Awesome-AI-Memory 中文 README](https://github.com/IAAR-Shanghai/Awesome-AI-Memory/blob/main/README_cn.md)，并结合本项目关注的 Memory VLA、部分可观测决策和长期机器人操作进行了重组与扩展。

## 1. 基本定义

大模型记忆可分为两类：

- **参数化记忆**：训练过程中内化到模型权重中的知识、能力与先验。它无需显式检索即可被输入激活，但更新成本高，来源和时效较难追踪。
- **显式记忆**：推理或交互过程中写入模型外部，并在需要时检索回上下文或策略网络的信息。它易于增删、追踪和设置访问边界，但效果依赖写入、检索和融合质量。

两者并非互斥。完整系统通常以参数化记忆提供通用能力，以显式记忆保存任务状态、用户偏好、环境变化和过去经验。对机器人而言，还需要考虑模型内部的递归状态、KV cache、对象地图等介于普通上下文和外部数据库之间的运行时状态。

## 2. 记忆系统的四层架构

| 层次 | LLM Agent 中的典型实现 | 机器人/Memory VLA 中的对应实现 | 核心问题 |
|---|---|---|---|
| 存储层 | 向量数据库、图数据库、文档库、混合索引 | 帧或 token bank、对象地图、场景图、关键帧库、轨迹库、递归状态 | 存什么、保存多久、容量如何增长？ |
| 处理层 | embedding、摘要、分块、实体与事件抽取 | 视频压缩器、事件检测器、对象跟踪器、任务进度提取器、动作编码器 | 如何把连续多模态流转换成可复用记忆？ |
| 检索层 | 召回、过滤、重排序、上下文注入 | 相似度检索、交叉注意力、门控读取、状态传播、空间查询 | 在何时读取哪些历史，并如何注入控制策略？ |
| 控制层 | 优先级、遗忘、冲突仲裁、访问控制 | 写入门、替换策略、时间衰减、状态修正、安全约束、算力预算 | 如何避免污染、陈旧信息和实时控制超时？ |

这一分层有助于区分“采用了向量数据库”和“形成了有效记忆系统”：存储介质只是底层组件，能否稳定提升任务表现取决于完整的写入—维护—读取—使用闭环。

## 3. 原子记忆操作

| 操作 | 通用定义 | 机器人系统中的例子 | 需要评估的风险 |
|---|---|---|---|
| 写入 | 将当前交互转换为可持久化记录 | 保存关键帧、对象状态变化、失败原因或语言纠错 | 冗余写入、错误观察固化、写入延迟 |
| 检索 | 根据当前状态召回相关记忆 | 找回被遮挡对象的位置、相似示范或先前子任务结果 | 漏召回、错误召回、历史干扰 |
| 更新 | 修订已有记录而非简单追加 | 更新对象位置、任务阶段或用户偏好 | 新旧证据冲突、身份错配 |
| 删除 | 主动移除特定记录 | 删除过期地图、错误经验或隐私数据 | 误删关键证据、残留副本 |
| 压缩 | 合并多条记忆并保留任务相关信息 | 将视频历史压缩为事件、语言摘要或递归状态 | 关键细节丢失、摘要幻觉 |

论文分析应明确方法实际支持哪些操作。只读历史窗口通常仅实现隐式写入和读取；具备 Add/Update/Delete、冲突处理和压缩机制的方法才接近完整的可管理记忆系统。

## 4. 生命周期与记忆管理

一个可部署的记忆生命周期可描述为：

1. **形成**：从观测、动作、语言和反馈中识别候选记忆。
2. **活跃使用**：将近期、高价值信息保留在低延迟工作记忆中。
3. **巩固**：去重、聚类、摘要或抽象为可跨任务复用的知识与技能。
4. **冷存储与再激活**：将低频记忆移至低成本存储，在相关条件出现时召回。
5. **修订或遗忘**：根据新证据、时效、隐私要求和资源预算更新或删除。

管理层至少应考虑：

- **冲突解决**：结合时间戳、来源可信度、感知置信度和物理一致性仲裁矛盾记录。
- **资源预算**：限制 token、帧数、显存、磁盘、网络带宽和单控制周期延迟。
- **一致性维护**：保证对象身份、空间坐标系、任务阶段和动作结果在不同模态间一致。
- **安全治理**：隔离用户、机器人和任务作用域，识别敏感信息，并防止不可信内容污染长期记忆。

## 5. 多维记忆分类

单一的“短期/长期”划分不足以描述机器人记忆。建议同时记录以下属性：

### 5.1 按时间范围

- **工作记忆**：当前控制周期或当前子任务使用的活跃状态。
- **情景记忆**：特定 episode、示范或失败过程的时序经验。
- **长期记忆**：跨 episode 保留的环境知识、用户偏好、程序和技能。
- **时效性记忆**：只在特定时间窗有效的动态状态。

### 5.2 按内容与结构

- **非结构化记忆**：原始文本、图像、视频或轨迹。
- **半结构化记忆**：对话摘要、关键帧、事件片段或语言化任务进度。
- **结构化记忆**：对象记录、空间地图、场景图、状态转移或技能图。
- **潜在记忆**：模型内部 memory token、KV state、RNN/SSM hidden state 或 fast weights。

### 5.3 按作用域

- **Episode 内**：解决遮挡、状态混叠、计数和任务进度问题。
- **跨 Episode**：复用示范、纠错、成功经验和环境规律。
- **个体级**：绑定特定用户、机器人或环境。
- **团队/公共级**：在多机器人或共享知识库中协同使用。

### 5.4 按功能

- **感知记忆**：保存过去视觉、力觉、本体感觉或空间状态。
- **程序记忆**：保存动作序列、技能和操作规则。
- **语义记忆**：保存对象属性、概念和稳定事实。
- **前瞻记忆**：保存未来需要执行的意图、约束或未完成目标。

## 6. 核心机制

### 6.1 检索增强

典型流程包括候选召回、上下文过滤、重排序和融合。索引可以是稠密向量、稀疏关键词、多向量、图结构或空间索引。机器人系统还需将任务相关性与时间、空间、对象身份和动作可行性联合建模，不能只依赖文本语义相似度。

### 6.2 反思与巩固

系统定期从原始轨迹中提炼失败原因、成功程序或环境规律。反思生成的内容不能天然视为事实，应保留来源轨迹、验证状态和置信度，并通过后续执行结果校正。

### 6.3 记忆路由

根据查询或控制状态选择工作记忆、情景库、语义地图、技能库或公共知识库。路由器需要同时优化检索质量和实时成本，并在不确定时支持多源召回或安全退化。

### 6.4 压缩与遗忘

- **内容级**：摘要、事件抽取、关键帧选择。
- **表征级**：token 压缩、量化、降维或固定容量状态。
- **组织级**：去重、聚类、分层存储和知识图谱巩固。
- **策略级遗忘**：时间衰减、低价值淘汰、冲突驱动替换和隐私删除。
- **参数级遗忘**：机器遗忘或模型编辑；它与删除外部记忆是不同问题，应分别报告。

## 7. 从 LLM 记忆迁移到机器人记忆时的关键差异

| 差异 | 对设计与评测的影响 |
|---|---|
| 连续且多模态的输入流 | 需要在线分段、跨模态同步和高频压缩，而非只处理离散对话轮次 |
| 记忆参与闭环动作 | 错误召回可能直接导致碰撞、抓取失败或任务状态破坏 |
| 物理世界持续变化 | 记忆必须区分稳定事实与动态状态，并支持观测驱动的修订 |
| 严格实时预算 | 检索、重排序和融合必须满足控制频率，平均延迟不足以描述最坏情况 |
| 坐标系与对象身份 | 语义相似并不能保证空间一致或指向同一实体 |
| 因果与可执行性 | 与当前查询相关的历史未必对动作有用，需要评估 action relevance |

因此，LLM 记忆基准上的问答准确率不能替代机器人闭环成功率。反之，单一完整任务成功率也无法定位写入、检索、推理或控制中的具体失败。

## 8. 后续论文分析模板

对每篇机器人记忆论文，可以从以下问题继续分析：

1. **记忆边界**：是模型参数、上下文、内部状态还是外部存储？
2. **作用范围**：服务于 episode 内、跨 episode，还是终身学习？
3. **记忆内容**：保存视觉、动作、语言、力觉、对象、空间还是程序？
4. **写入机制**：固定频率、事件触发、重要性预测还是人工纠错？
5. **维护操作**：是否支持更新、删除、合并、衰减和冲突仲裁？
6. **读取机制**：全量注意力、相似度检索、图查询、门控还是递归传播？
7. **融合位置**：记忆进入高层规划、VLM backbone、action expert 还是安全控制器？
8. **容量复杂度**：存储、计算和延迟是否随 episode 长度增长？
9. **训练路径**：记忆是否可微，长程梯度如何传播，是否仅在推理时工作？
10. **可信性**：如何处理错误感知、冲突、陈旧状态、隐私和记忆投毒？
11. **评测有效性**：任务是否存在 state aliasing，是否设置 no-memory、wrong-memory 和 oracle-memory 对照？
12. **控制收益**：记忆改善的是检索准确率、规划质量、子任务完成率还是闭环任务成功率？

这些维度可作为本项目后续扩展论文表格、方法卡片和统一 benchmark 报告规范的基础。
